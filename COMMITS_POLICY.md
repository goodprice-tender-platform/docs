# Соглашение по сообщениям коммитов

Универсальная система пометок для сообщений коммитов помогает поддерживать порядок в истории изменений и облегчает навигацию по проекту. Данный документ описывает стандартизированные теги, которые можно использовать при оформлении коммитов.

## Основные теги

Ниже перечислены основные категории изменений:

- **`feat`** — добавление новой функциональности.
- **`fix`** — исправление ошибки.
- **`update`** — обновления (например, обновление зависимостей или библиотек).
- **`docs`** — изменение или добавление документации.
- **`style`** — правки, не влияющие на логику (форматирование, правки стилей).
- **`refactor`** — рефакторинг без изменения поведения программы.
- **`test`** — добавление или изменение тестов.
- **`chore`** — различные служебные изменения (настройки CI/CD, конфигурационные файлы и др.).
- **`wip`** — *Work In Progress* — используется, когда работа продолжается, но ещё не завершена.

## Формат сообщения

Рекомендуемый формат коммита:

```<тег>(<компонент>): <краткое описание>```

### Примеры

```
feat(auth): add login functionality
fix(api): resolve crash on data fetch
update(deps): bump react to v19
docs(readme): add setup instructions
style(css): improve layout spacing
refactor(core): simplify calculation logic
test(auth): add tests for login flow
chore(ci): update pipeline configs
wip(profile): working on profile page layout
```
